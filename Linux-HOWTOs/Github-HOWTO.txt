============
Github HOWTO
============

Créer un compte sur https://github.com


Configurer Git 
--------------

Nom de l'utilisateur :

  $ git config --global user.name "Kiki Novak"

Adresse mail :

  $ git config --global user.email "info@microlinux.fr"

Garder l'identifiant et le mot de passe en cache pour éviter d'avoir à les
taper à chaque modification :

  $ git config --global credential.helper cache

Définir la durée d'expiration. Une semaine par exemple :

  $ git config --global credential.helper 'cache --timeout=86400'


Créer un dépôt
--------------

Créer un dépôt sur Github, par exemple :

  · Owner           : kikinovak

  · Repository name : documentation

  · Description     : La documentation Linux de Kiki Novak
  
  · [X]               Public

  · [ ]               Initialize this repository with a README

  · Add .gitignore  : None

Cliquer sur [Create Repository].

Sur la machine locale :

  $ mkdir documentation
  $ cd documentation

Éditer un fichier 'documentation/README.md' comme ceci par exemple :

--8<---------- README.md -----------------------------------------------------
Documentation
-------------

Ce dépôt contient la documentation Linux de Kiki Novak, sous forme d'une série
de fichiers HOWTO basés essentiellement sur la distribution Slackware Linux.
--8<--------------------------------------------------------------------------

  $ git init

  /home/kikinovak/Documents/Github/documentation/.git/

  $ git add README.md

  $ git commit -m "Commit initial."

   create mode 100644 README.md

Afficher le statut
------------------

  $ git status

  nothing to commit (working directory clean)

Si l'on modifie un fichier :

  $ git status
  # On branch master
  # Changes not staged for commit:
  #   (use "git add <file>..." to update what will be committed)
  #   (use "git checkout -- <file>..." to discard changes in working directory)

  #
